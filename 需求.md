# jjlove 智能点单系统 - 需求文档

## 一、 产品定位
一款专为情侣/夫妻设计的轻量级、充满生活情趣的“专属点单与代购”移动端 Web 应用。
核心链路：**点单 -> 线下采购 -> 记录完成**（无需真实的支付结算功能）。

## 二、 核心角色与登录
* **角色设定**：分为“点单方”（干饭王）和“采购方”（投喂官）。（名称可自定义修改）
* **极简登录**：无需复杂注册。移动端打开后，展示两个专属头像，点击自己头像输入 4 位专属 PIN 码即可进入。
* **状态标识**：界面顶部清晰标记当前登录的是哪一方。

## 三、 核心功能模块

### 1. 菜单与分类管理 (Menu & Categories)
* **动态分类**：默认提供 **早餐、晚餐、零食、日用品** 四大分类。分类支持后台/前端编辑（增删改），方便后期拓展。
* **二级分类**：每个主分类下支持子分类（如早餐->饮品/主食，零食->甜食/咸食）。
* **商品管理**：支持添加、编辑商品。商品信息包含：
  * 名称
  * 图片（支持手机拍照或相册上传）
  * 价格（预估价格，用于计算总额）
  * 所属分类

### 2. 点单流程 (Order Flow)
* **选品购物车**：点单方在移动端浏览分类菜单，通过“+ / -”按钮选择商品。
* **底部结算栏**：实时显示已选商品数量和预估总金额。
* **提交订单**：确认无误后点击“发送订单”，订单进入“待采购”状态。

### 3. 采购与核销流程 (Purchase & Check-off)
* **任务清单**：采购方登录后，首屏展示“待处理订单”。
* **交互式打勾**：采购方在线下超市/餐厅购买时，买到一件勾选一件（附带划线效果和可爱音效/动效）。
* **订单完成**：全部勾选后，点击“完成采购”，订单归档至历史记录。

## 四、 趣味与情感化设计（进阶建议）
*(注：以下为增加情趣的扩展功能，可分阶段实现)*
1. **“随便/盲盒”模式**：点单时如果不知道吃啥，可以点一个“盲盒”，让采购方自己决定买什么，制造小惊喜。
2. **“饿饿饭饭”一键催单**：点单方等急了，可以点击一个“催单”按钮，界面上会有可爱的动画（后期可接入微信/Server酱通知）。
3. **心愿币/跑腿费系统**：采购方每次完成订单，可以获得虚拟的“心愿币”。心愿币可用于兑换“免洗碗券”、“捶背10分钟”等情侣互动小奖励。
4. **投喂回忆墙（数据统计）**：记录两人使用的天数、累计点单次数、最爱吃的菜品Top3等，生成专属的“投喂账单”。

## 五、 技术架构
* **前端**：React 18 + Tailwind CSS 3 + Vite。采用移动端优先PWA设计，支持添加到主屏幕和全屏模式。
* **后端**：**Node.js** + Express 提供轻量级 RESTful API。
* **数据库**：SQLite（单文件数据库，极其轻量，无需额外部署，非常适合个人/情侣项目）。
* **部署**：阿里云 ECS，使用 Docker + Nginx 部署。

## 六、 UI/UX 风格
* **Style C - 奶油小猪/Kmall风格**：
  * 主色调：温暖奶油黄 (#FFF8DC)
  * 强调色：淡粉色 (#FFB6C1) 
  * 边框：粗黑线描边，卡通风格
  * 圆角按钮，圆润字体，可爱图标
* 移动端优先设计，支持PWA（可添加到主屏幕，全屏模式运行）
* 微交互动效：点击完成时的撒花效果、可爱音效
* 跨平台兼容：Android和iOS设备均可安装使用
